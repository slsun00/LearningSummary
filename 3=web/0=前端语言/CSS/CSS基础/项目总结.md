## 起类名

```css
头部 header
导航栏 nav
搜索框 search
个人信息 user
侧边栏  subnav

导航栏
	实际开发中不会直接使用链接 a , 而是 li , 包含链接 (li+a) 的做法
	li+a 语义更加清晰，有条理的列表型内容，
	a 搜索引擎容易辨别为对切关键字嫌疑，会被降权


模块
	父模块和字模块，通过设置固定的长度，使其正好可以符合完全布满父模块
```

## 居中固定

```css
1. 固定到可视页面的固定位置
2. 固定在版心右侧
	/*先走到页面中间，即版心位置，再走版心盒子宽度的一半*/
	1. 先固定定位盒子left50% 走到浏览器可视区(版心)的一半的位置
	2. 让固定定位的盒子 margin-left : 版心宽度的一般距离，夺走版心宽度的一半位置
	固定定位的盒子贴着版心右侧对齐

3. 绝对定位的盒子居中
.box {
    position:absolute;
    /* left 走 50% 父容器宽度的一半 */
    left:50px;
    /* margin 负值 往左走自己盒子宽度的一半*/
    margin-left:-100px;
}
```
