## 事件

### 介绍

```js
方式一
	// eventName(fn);
	$("button").click(function(){});
	效率高，但是部分事件 jQuery 没有实现，所以不能添加
方式二
	// on(evenName, fn); 主流
	$("button").on("click", function(){})
	效率低，所有 js 事件都可以添加
```

### 阻止冒泡

```js
冒泡
	儿子响应事件后， 父亲也会响应自身的事件
阻止冒泡
// 注意写到儿子当中
方式一
	$(".son").click(function(){
		return false;
    })
方式二
	$(".son").click(function(event){
        event.stopPropagation()
    })
```

### 阻止默认行为

```js
默认行为
	事件自带的行为，点击连接会跳转
阻止模型行为
方式一
	$("a").click(function(){
		return false;
    })
方式二
	$("a").click(function(event){
        event.preventDefault();
    })
```

### 自动触发事件

```js
trigger(eventName,function(){}); //  会触发冒泡、默认行为，可以自定义事件
trggerHandler()		// 不会触发冒泡、默认行为
```

### 自定义事件

```js
// on 绑定 ， trigger 触发
$("son").on("myClick",function(){});
$("son").trigger("myClick");

// 使用，进行 命名空间
// 
$("son").on("click.zs",function(){})
$("son").trigger("click.zs");
注意
	1. 触发子元素命名空间的事件，父元素相同命名空间的事件也会触发，不相同的不触发    
    2. 触发子元素不带命名空间的事件，父类所有相同类型的事件也会被触发
```

### 事件委托

```js
介绍
	自己不做，让别人做，做完的结果返回给自己
    把事件委托给，入口函数执行前就有的元素
语法例子
	// 把 li 的点击事件，委托给 ul 进行监听
	$("ul").delegate("li","click",function(){})
注意
	现在基本都使用 on 进行委托
```
