## 坐标



### 使用

```java
概述
    你在使用哪个项目的那个包，是需要使用哪个 api ，是需要你记忆的
    原因
    	1. 可能不同项目或者框架的 api 设置的是一样的，
    		你如果没有记忆清楚使用的是什么项目的，很有可能就会导错包
        // spring 框架的导包都是有深思熟虑的，感觉真的是高！！！！
使用例子
        <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-context</artifactId>
    </dependency>
    // 像这种的就是 spring 框架有意为之，让groupId 和 包名一致
    import org.springframework.web.bind.annotation.RestController;
    
```



### 解读

```java

    
坐标结构
   <groupId>com.boyan</groupId>	
   <artifactId>common</artifactId>            
项目中的结构
   包结构：common/src/main/java/  com/learn/slsun/exception/ExceptionUtils.java          

使用 
   import com.learn.slsun.exception.ExceptionUtils;  

总结
    你在项目汇总的 java 目录中使用的 import 语句，
    基本上是无法判断出来他这个包是属于哪个项目的，除非是项目发起者是有意为之
    你能够从项目中看出来是使用的那个包，应该是作者有意为之，像上面我写的，就是八竿子打不着的关系....
    /*
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-context</artifactId>
    </dependency>
    // 像这种的就是 spring 框架有意为之，让groupId 和 包名一致
    import org.springframework.web.bind.annotation.RestController;
    
    */
```



### 寻找

```java
从 import 寻找坐标
    import javax.servlet.http.HttpServletRequest;
1. 复制 javax 到 https://mvnrepository.com 中去寻找
2. 根据自己的印象，去引入具体的坐标
    对，就是根据你的印象，因为你使用哪个包，需要最起码对他所属的坐标有印象
```
