## 环境变量配置

### 过程

```java
将 maven 解压出来的目录的路径
    添加到环境变量中
    
环境变量： 新建
    变量名：MAVEN_HOME
    变量值：maven 解压文件放置的路径
    
系统变量： path
    新建，然后添加
    %MAVEN_HOME\bin%，注意百分号不能少，
    就是引用到了新建的MAVEN_HOME之下的bin目录下
    
```

### 验证

```java
cmd 命令行  
    mvn -v
    
    出现以下类似的信息就是安装成功
    //Apache Maven 3.6.3 (cecedd343002696d0abb50b32b541b8a6ba2883f)
    
```

## 本地仓库配置

```xml
介绍
    Maven默认的本地仓库地址在我们的电脑C盘用户目录
    外国的远程仓库又在国外，下载速度肯定慢死
    
修改
    maven 安装目录下的 conf 文件夹
    在 %M2_HOME%\conf 目录中的 Maven 的 settings.xml 文件中定义另一个路径。
    运行 Maven 命令，Maven 将下载依赖的文件到你指定的路径中   

    <!--
        config/settings.xml
    -->
	<settings ... >
	 <!-- 注意是 / -->
      <localRepository>C:/MyLocalRepository</localRepository>
          
</settings>
```




### 阿里云配置

```java
// 1:修改 maven 根目录下的 conf 文件夹中的 settings.xml 文件，在 mirrors 节点上，添加内容

// %M2_HOME%\conf\settings.xml , 放在 mirro
<mirrors>
    <mirror>
      <id>alimaven</id>
      <name>aliyun maven</name>
      <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
      <mirrorOf>central</mirrorOf>        
    </mirror>
</mirrors>
          
          
          
// 项目中添加          


// 2: pom.xml文件里添加
<repositories>  
     <repository>  
        <id>alimaven</id>  
        <name>aliyun maven</name>  
        <url>http://maven.aliyun.com/nexus/content/groups/public/</url>  
        <releases>  
            <enabled>true</enabled>  
        </releases>  
        <snapshots>  
            <enabled>false</enabled>  
        </snapshots>  
    </repository>  
</repositories>
                    
```
