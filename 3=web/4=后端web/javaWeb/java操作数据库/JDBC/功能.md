## 事务管理

### 介绍

```java
介绍
    一起执行的代码块
执行
    开启 、提交、 回滚
使用
    try {
		// 数据库其他操作， sql 执行之前开启
        conn.setAutoCommit(false);
        //其他数据操作，操作结束后提交
        conn.commit();
    }catch (Exception e){
        if (conn != null){
            conn.rollBack(); 
        }
       
    }finally{
		// 关闭
    }
```

## 数据库连接池

### 基础

```java
介绍
    // 用来连接数据库的
    一个容器（集合），存放数据连接的容器
    当系统初始化以后，容器被创建，容器中会申请一些连接对象
    	当用户来访问数据库时，从容器中获取连接对象
    	当用户访问完以后， 会将连接对象归还给容器
优势
    节约资源
    用户访问效率高
实现
    标准接口 DataSource javax.sql 包中
    获取连接：getConnection()
    归还连接：如果连接对象是从连接池中获取，调用 connection.close()方法，就不会关闭连接，而是归还
注意
    这个而一般是由数据库厂商实现
技术实现
    C3P0
    Druid
```

### C3P0

```java
使用
    导包:两个包 c3p0-0.9.5.2.jar  mchange-commons-java-0.2.12.jar
    定义配置文件 ： 
        名称： c3p0.properties c3p0-config.xml
        路径： 直接放在 src 目录下
    创建对象： 数据库连接对象 ComboppoledDataSoutce
    获取连接： getConnection
配置
    可以配置不同的配置文件，但是普遍还是默认的
```

### Druid

```java
使用
    导包:druid-1.00.0.jar
    定义配置文件 ： 
        名称： druid.properties （名称任意）
        路径： 可以放在任意的文件目录下 ，需要手动配置
    加载配置文件 Properties
    创建对象： 数据库连接对象，通过工厂来获取 BuildDataSourceFactory
    获取连接： getConnection
配置
    可以配置不同的配置文件，但是普遍还是默认的
```
