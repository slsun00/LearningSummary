## 建表规约

### 数据类型

```java
小数类型
    为 decimal，禁止使用 float 和 double，建议将数据拆成整数和小数并分开存储。
字符串
    存储的字符串长度几乎相等，使用 char 定长字符串类型。
可变长字符串
    varchar 
    	不预先分配存储空间，长度不要超过 5000
    text	
    	长度超过 5000 使用，
    	独立出一张表，用主键来对应，避免影响其它字段索引效率
    
字符长度
    // 合适的字符存储长度，不但节约数据库表空间、节约索引存储，更重要的是提升检索速度   
    对象 年龄区间 类型 字节 表示范围
	人 150 岁之内 tinyint unsigned 1 无符号值：0 到 255
```

### 字段

```java
表达是否概念
    命名: is_xxx 方式命名
    类型： unsigned tinyint（1 表示是，0 表示否）。  
    例子： 逻辑删除的字段名 is_deleted，1 表示删除，0 表示未删除 
        
主键索引名
    pk_字段名；即 primary key
唯一索引名
    uk_字段名； 即 unique key
普通索引名则
    idx_字段名，即index 的简称。        
        
必备字段
    id 必为主键，类型为 bigint unsigned、单表时自增、步长为 1。
    gmt_create, datetime 类型    现在时表示主动式创建
	gmt_modified， datetime 类型  过去分词表示被动式更新

更新
	修改字段含义或对字段表示的状态追加时，需要及时更新字段注释
冗余字段
	// 字段允许适当冗余，以提高查询性能，但必须考虑数据一致
        1） 不是频繁修改的字段。
        2） 不是唯一索引的字段。
        3） 不是 varchar 超长字段，更不能是 text 字段。
     例子 
        各业务线经常冗余存储商品名称，避免查询时需要调用 IC 服务获取
```

### 表

```java
命名
    1. 不使用复数名词，其仅表示实体内容，不表示数量
    2. 遵循“业务名称_表的作用”	guli_edu
数据过多
    单表行数超过 500 万行或者单表容量超过 2GB，才推荐进行分库分表。
    如果预计三年后的数据量根本达不到这个级别，请不要在创建表时就分库分表。
```
