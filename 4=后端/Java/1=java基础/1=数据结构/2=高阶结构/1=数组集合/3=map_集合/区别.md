## hashMap & hashTable

### 相同点

```java
1. 都是基于哈希表实现的，其内部每个元素都是 key-value 键值对，
2. 都实现了 Map、Cloneable、Serializable 接⼝。    
```



### 不同点

```java
1. 父类不同
	HashMap 继承了 AbstractMap 类，
    HashTable 继承了 Dictionary 类    
    
2. 空值不同
    HashMap 允许空的 key 和 value 值，
    	会把 Null key 当做普通的 key 对待
    	不允许 null key 重复
    HashTable 不允许空的 key 和 value 值。
    	如果 key 或者 value 为 null ，则抛出异常
    
3.线程安全
    HashMap 不是线程安全的
    HashTable 本身就是线程安全的容器
    
4. 性能⽅⾯
    HashMap 和 HashTable 都是基于 单链表
     HashMap 进⾏ put 或者 get操作，可以达到常数时间的性能；
    HashTable 的 put 和 get 操作都是加了 synchronized 锁的，所以效率很差。
    
5. 初始容量不同
    HashTable 的初始⻓度是11，之后每次扩充容量变为之前的 2n+1（n为上⼀次的⻓度）
    HashMap 的初始⻓度为16，之后每次扩充变为原来的两倍
    创建时，如果给定了容量初始值，
    	那么HashTable 会直接使⽤你给定的⼤⼩，
    	⽽ HashMap 会将其扩充为2的幂次⽅⼤⼩
```



## HashMap 和  HashSet

### 不同点

```java
HashSet 
    继承于 AbstractSet 接⼝，
    实现了 Set、Cloneable,、java.io.Serializable 接⼝
    
HashSet 
    不允许集合中出现重复的值。    
    底层其实就是 HashMap，所有对 HashSet 的操作其实就是对 HashMap 的操作
     HashSet 也不保证集合的顺序，也不是线程安全的容器
```
