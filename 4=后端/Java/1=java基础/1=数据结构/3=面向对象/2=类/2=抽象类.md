

## 抽象类

### 介绍

```java
接口
    定义一种规范
抽象类
    定义一种规范的同时，同时还可以操纵一系列类，
```



### 抽象方法 

```java
 介绍
     父类中的方法 ，不确认如果进行 {} 方法体的实现， 这个方法就是抽象方法
     抽象方法只能存在抽象类中
语法格式
     abstract 返回值 方法名（参数）；// 没有方法体， 没有大括号
     abstract void fun()；
```

### 抽象类

```java
介绍
	抽象方法所在的类
类中方法
    抽象方法
    抽象类

使用
    1. 抽象类不能实例化， 不能创建 new 抽象类对象
    2. 必须使用一个子类去继承
    3. 子类必须覆盖重写（实现）父类中的所有方法 ， 否则无法编译通过
    	子类去掉 abstruct 关键字， 并补全自己需要的方法体
  	4. 创建子类对象进行使用
注意
    抽象类可以有构造方法 ， 供子类创建对象的时候， 初始化父类成员使用
    如果抽象类的子类也是抽象类， 那子类就不需要实现父类的方法
    	抽象子类的
```

### 例子

```java
// 抽象类
public abstruct class Animal {
    // 抽象方法
    public abstruct void eat();
}

public class Cat extends Animal {
    @Override
    public void eat() {};
}
```

### 纯抽象类

```java
相当于接口， 方法都没有实现
```

### 思辨

#### 和接⼝的区别

```java
不同点
1. 抽象级别不同
    抽象程度依次是 接⼝ > 抽象类 > 类

3. 关键词
    类使⽤ class 来表示；
    抽象类使⽤ abstract class 来表示；
    接⼝使⽤  interface 来表示
        
2. 方法
    // ⽅法的定义是不允许在⽅法后⾯出现 {}
    接⼝： 只允许进⾏⽅法的定义，不允许有⽅法的实现
    抽象类： 可以进⾏⽅法的定义和实现
    类中： 只允许进⾏⽅法的实现  
        
4. 变量
	接⼝中定义的变量只能是公共的静态常量，
    抽象类中的变量是普通变量。        
```
