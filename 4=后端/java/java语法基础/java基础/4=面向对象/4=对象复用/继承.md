

## 继承

### 介绍

```java
优势
    可以减少重复的代码
    子类可以拥有父类的成员
    子类还可以拥有自己专属的成员
名词
    子类 、派生类（subclass）。
    父类 、基类、超类（super class）

特点
    // 接口之间也有继承
    类与类    ：
    	单继承， 直接父类只有一个
	    多重继承，
    		一个类可以有父类(直接父类)、爷爷类(间接父类)、曾祖类(间接父类) 
    类与接口   ：
    	多实现 ， 一个类可以实现多个接口
    接口与接口  ：
    	多继承 ，一个接口可以继承多个接口


```



### 语法格式

```c++
关键字
    extends
语法
	public class son extends father {}
 方式
     // 静态 、 非静态都是一样的
    // 注意对于跨包的，需要再说
     子类完全继承父类的属性、方法， 即使是 private 修饰的
     父类声明为 private 的属性和方法，子类继承后， 仍然可以认为获取了父类中私有的结构， 只是因为封装性的影响，使得子类不能直接调用父类的结构而已

	public 		子类完全继承 =》 跨包可用

    default		 子类完全继承 =》 本包
    private      子类不能继承 =》 类内部
```

### 构造顺序

```c++
介绍
     // 子类继承父类以后， 当创建子类对象，也会调用父类的创建函数
    子类构造方法默认有一个隐藏的的 super() 调用
    所以 ： 先调用父类的构造函数，再调用子类的构造函数
特点
    可以通过 super() 来调用父类的重载构造
        super()  // 调用无参构造
        super(参数) // 调用父类有参构造
	super() 必须卸载子类的第一个语句， 不能多次调用 super 构造
    
总结
    子类继承父类， 必须调用父类的构造方法
    	不写 ：就赠送 super()
    	写了 ： 写了则写的指定的 super 调用， 但是只能调用一次
```

## 复用冲突

#### 介绍

```c++
介绍
    子类和父类出现同名的成员
    静态成员和非静态成员出现同名，处理方式一样
相关概念
    重名
        成员变量 ： 重名, 不能重写，区别访问
        成员方法 ： 覆盖重写
    重写
    	// 覆盖重写 override
    	方法的名称、参数列表都一样
     	返回值 ： 子类返回值范围  <= 父类返回值范围
    	修改符 ： 子类权限 <= 父类权限(基本是相等的权限)
   	重载
    	方法的名称一样 ， 参数列表不一样
变量重名
    1. 通过对象名称进行访问： 看等号左边是谁， 就优先用谁，没有则向上找
    	// obj 进行变量访问，先从 father 类中找
    	Father obj = new Son()
    2. 通过成员方法进行访问 ： 方法属于谁，优先用谁 ，没有则向上查找
    	public class Son{
            int num = 10;
            public void show() {
                // show 方法属于 Son , 所以优先调用 Son 中的 num
                num += 1;
            }
        }
方法重名
    总原则
    	无修饰符 ：如果没有就想上找父类 ， 绝对不会向下找子类
    	有修饰符 ： 使用限定的对象 ，this 自身 ，super 父类
    // 成员变量、方法
    	父类实例 ： 就使用自己
		子类实例： 子类有就使用子类的 ，子类没有就想上找父类的， 倒出都没有就报错

注意
    如果子类中出现和父类同名的成员函数， 子类的成员成员会自动隐藏掉父类中所有的同名成员函
```

## 纯继承 

```java
介绍
    将抽象类设置为类似接口的形式
    方法都要是 abstract 方法 , 字段无值
特点
    类似接口
    纯粹 is - a 的关系，确保所有
缺点
    一旦向上转型， 就不能使用扩展类新增的部分， 只能使用基类的含有的方法 
```
