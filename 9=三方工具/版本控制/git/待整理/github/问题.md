# 删除

## 文件

### 本地仓库删除

```css
git rm 文件名称
git rm -r 文件夹/


git rm <file>

// 添加参数
-f 
    强制删除
    删除之前有修改过并且已经放到暂存区域操作
--cached
    文件从暂存区域移除，但工作区保留
    即仅是从跟踪清单中删除
-r
    git rm -r // 会删除执行语句目录下的所有文件和子目录
```



###  github 库

*   删除文件

    ```css
    1. 在 github 直接可以删除
    2.  仅仅删除远程分支文件，不删除本地文件
        git rm --cached filename
        git commit -m "delete remote file filename "
        git push -u origin master(此处是当前分支的名字)
    3. 删除本地文件与远程分支文件
    	git rm filename
        git commit -m "delete file filename "
        git push -u origin master(此处是当前分支的名字)
    ```

*   删除文件夹

    ```css
     /* 在github上只能删除仓库,却无法删除文件夹, 所以只能通过命令来解决 */
    方法一

                    -r表示递归所有子目录，如果你要删除的，是空的文件夹，此处可以不用带上-r。
    ```

# 拉取

## 拉取某一个分支的指定文件、文件夹

```css
方法一 ：此方法适用于 Git1.7.0 以后版本，，本质是开启sparse clone


git remote add -f origin http://github/projectName.git 
		# 添加远程仓库地址，实现拉取remote的all objects信息

		
git config core.sparsecheckout true
        # .core.sparsecheckout用于控制是否允许设置pull指定文件/夹，true为允许。
		# 开启sparse clone
	
echo "fileName/*" >> .git/info/sparse-checkout
	   # 在.git/info/sparse-checkout文件中（如果没有则创建）添加指定的文件/文件夹
	   # 就是需要拉取的那个特定文件/夹,设置需要pull的目录，*表示所有，!表示匹配相反的
	
git pull origin master
	    # 拉取指定目录
	    # 拉取命令是一样的，只是已经通过配置文件sparse-chechout指定了目标文件/夹
	    # 将origin 端，由第三步（文件 .git/info/sparse-checkout）设置的 目录下的文件 pull 到本地	
```
