## 启动服务

```java
介绍
    svn 服务器必须处于运行状态才能响应客户端请求，进行管理项目文件
// svnserve -d -r 目录(绝对路径) --listen-port 端口号
介绍
    -d 表示后台执行
    -r 表示版本库的根目录
    由于-r 配置方式的不一样，SVN启动就可以有两种不同的访问方式
    
    -r: 配置方式决定了版本库访问方式。

        方式二： 
    --listen-port: 指定SVN监听端口，不加此参数，SVN默认监听3690
        
检测执行成功
	cmd 中执行 netstat -an   // 3690 被监听就表示服务已经启动
        
命令行关闭
	每次启动 svn 命令行窗口，都要保持，不能关闭        
```

### 方式一：-r

```java
方式一： -r直接指定到版本库(称之为单库svnserve方式)
    svnserve -d -r /opt/svn/runoob
    
注意
    在这种情况下，一个svnserve只能为一个版本库工作
    authz配置文件中对版本库权限的配置应这样写：
    [groups]
    admin=user1
    dev=user2
    [/]
    @admin=rw
    user2=r
    // 使用类似这样的URL：svn://192.168.0.1/　即可访问runoob版本库
```

### 方式二： -r

```java
指定到版本库的上级目录(称之为多库svnserve方式)
    svnserve -d -r /opt/svn
    
一个svnserve可以为多个版本库工作
     authz配置文件中对版本库权限的配置应这样写：
    [groups]
    admin=user1
    dev=user2
    [runoob:/]
    @admin=rw
    user2=r

    [runoob01:/]
    @admin=rw
    user2=r
    // 如果此时你还用[/]，则表示所有库的根目录，同理，[/src]表示所有库的根目录下的src目录。
	// 使用类似这样的URL：svn://192.168.0.1/runoob　即可访问runoob版本库。
```

### 方式三：win服务注册

```java
介绍
    关闭命令行窗口，服务也在启动
    
命令
    
    // windows 服务注册, 注意格式,= 后面有一个空格， 需要用管理员身份打开
    
    // binpath 可执行文件的路径  svnserve.exe 的路径， svn 安装路径/bin/svnserve.exe
    sc create SVNService binpath= "E:\svn\bin\svnserve.exe --service -r 版本库路径" start= auto depend= Tcpip
    
    参数
    
```
