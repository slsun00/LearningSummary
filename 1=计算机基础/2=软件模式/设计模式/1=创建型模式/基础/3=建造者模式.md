## 模式结构

```java
// 抽象建造者 interface 建造者 class
	// (零部件) + 空对象(整体)
	只负责创建对象，不负责创建对象的顺序： 创建
    
// 指挥者
    // 执行对象组装填充
	负责创建对象的顺序： 顺序
    
// 产品类
	创建的对象
// 运行逻辑
	建造者：
        将车子的零件组装好，相当于工厂，只负责生产，不负责生产的顺序
	指挥者：
        内嵌建造者，
        负责规定建造者的创建顺序
// 注意
	1. 抽象工厂，其实就是建造者，只是抽象工厂并不强调产品创建的顺序
	2. 指挥者的角色可以不要，可以直接写在指挥者内部的一个方法中        
        

// 角色定义
* 抽象建造者类（Builder）：
    这个接口规定要实现复杂对象的那些部分的创建，并不涉及具体的部件对象的创建。 

* 具体建造者类（ConcreteBuilder）：
    实现 Builder 接口，完成复杂产品的各个部件的具体创建方法。在构造过程完成后，提供产品的实例。 

* 产品类（Product）：
    要创建的复杂对象。

* 指挥者类（Director）：
    调用具体建造者来创建复杂对象的各个部分，在指导者中不涉及具体产品的信息，只负责保证对象各部分完整创建或按某种顺序创建。
```
